# Contributing to DIGITAL UNIVERSITY OF KERALA EXTRACTION MODEL

First off, thank you for considering contributing to this project! It's people like you that make it a great tool for the geospatial community.

## Code of Conduct
By participating in this project, you are expected to uphold our high standards of professional conduct and collaboration.

## How Can I Contribute?

### Reporting Bugs
- Use the GitHub Issue Tracker.
- Describe the bug in detail and provide reproduction steps.
- Include your environment details (OS, Python version, CUDA version).

### Suggesting Enhancements
- Open a new issue with a clear title and description.
- Explain why this enhancement would be useful for the project.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. Ensure your code follows the existing style (use `flake8` or `black`).
3. Add docstrings and type hints for all new functions.
4. Update the documentation (README, Architecture) if necessary.
5. Ensure the Streamlit app still runs correctly.

## Coding Standards
- **Python**: 3.8+
- **Style**: PEP8
- **Documentation**: Google-style docstrings.
- **Type Hints**: Mandatory for all new APIs.
